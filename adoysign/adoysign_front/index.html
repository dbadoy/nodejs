IPFS

탈중앙화된 저장공간





기존 구조에서 서버에 모든 데이터가 저장되었던 것과 달리, IPFS에서는 전 세계 여러 컴퓨터에 분산
저장됩니다.
데이터를 불러 올때는, 고유값(CID)으로 찾아 다수의 컴퓨터에서 조각조각 나누어 받아와 합칩니다.

IPFS는 중앙서버의 다운, 데이터 위변조 문제를 해결합니다.

Adoysign에서 문서, 서명 정보는 모두 IPFS, 블록체인 분산되어 저장됩니다.

※ Adoysign 서버가 다운되더라도 문서, 서명 정보는 남아있으며, 위변조 또한 불가능합니다.

How to use Adoysign ?

1
Step 1 - 회원가입
Adoysign을 이용하기 위해서는 EOS계정이 필요합니다.
회원가입시, EOS계정과 퍼블릭키, 프라이빗키를 입력해주세요.

※프라이빗키는 계정 인증에만 사용되며, 저장되지 않습니다.
EOS : Jungle3 TestNet
2
Step 2 - 문서등록
먼저, 서명을 원하는 문서를 등록하세요.
문서를 등록시 제공되는 문서번호와 CID를 상대방에게 제공하면 됩니다.
※내 정보에서 다시 확인할 수 있습니다.
예시)
문서번호 : 200000000000001
CID : QmTX89Rck6Tsnj3UwaL1AxVGMpiWH8h8bwjnDc2po9YVUY
https://gateway.ipfs.io/ipfs/(CID)
3
Step 3 - Signcode 생성
CID를 받은 사용자는 문서를 확인 후 서명을 하려고 합니다.
Adoysign에서는 서명 할 때 Signcode를 필요로 합니다.
Signcode는 일회용 비밀번호 입니다.
발급 받은 Signcode는 타인에게 보여줘서는 안되며, 복구가 불가능하기 때문에,
잘 보관하여야 합니다.
※서명 할 때마다 생성하여 사용하는 것을 권장합니다.
예시)
번호 : 100000000000001
Signcode : abcde
4
Step 4 - 서명확인
문서의 서명을 확인할 차례입니다.
"서명확인" 메뉴에 들어가 문서번호, CID 를 입력하면
해당 문서에 대한 서명을 확인 할 수 있습니다.
